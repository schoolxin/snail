7.2.Session

7.2.Session

学习目标

  • 理解 Session 的数据传递流程
  • 熟练的使用 Session 相关的函数

一、Session 概念

Session 本质上是保存在服务器端的key-value数据，Session 机制需要依赖于 cookie 机制

  • session数据可以保存在内存中，也可以保存在磁盘上，比如 mysql 数据库里
  • 保存在内存中的 session 数据在服务器断电之后，就会丢失

1.1 Session 数据的传递流程

以用户登录为例：

  1. 浏览器第一次访问服务器的时候，cookie 里面没有 session 相关信息，如果是用户登录，请求会携带用户名和密码的信息
  2. 当我们的服务器收到这个请求后，处理请求，返回 http 响应，并且会在响应中添加 set-cookie 响应头，里面返回 session_id 和 session_key 这个键值对
    1. session 中间件先从 cookie 里面获取 session_id 这个cookie的值，这个值就是 session_key，如果获取不到 session_key 就随机生成一个 32 位的由小写字母和数字组成的字符串作为 session_key
    2. 视图会进行用户名和密码验证，验证没有问题，一般会把 user_id 从用户表里面取出来，保存到 session 里面，以 'user_id' 为 key，数据库中查询到的 user_id 值为 value，存储在 session 里面
    3. 视图返回 response 后，session 中间件还会在 response 上设置一个 session_id 这个 cookie 值，这个 cookie 值的 key 是 ‘session_id’，value 是第1步产生的 session_key
    4. session 中间件把 session 这个字典数据保存在数据库中，session_key 是主键的值，session 字典转换为字符串后保存为 data 的值
  3. 在浏览器收到响应数据后，会从 cookie 里面获取到 session_id 这个 cookie 的 key-value，并且保存起来
  4. 浏览器第二次访问服务器的时候，cookie 里面就自动带上 session_id 这个 cookie 的 key-value
  5. 服务器第二次收到请求后，就可以从 cookie 里面获取到 session_id 这个 key-value，也可以得到前面的 session_key，然后从数据库中加载查询 session 的数据，并且转换为字典

二、Session的配置和api

2.1 启用Session中间件

Django 框架通过中间件提供了 Session 功能的支持，如下图所示，在 settings.py 文件中，默认 Session 中间件是启用了的，如需禁用 session，将 Session 中间件注释掉即可。

2.2 配置Session存储方式

在 settings.py 文件中设置 SESSION_ENGINE 常量，可以设置 session 数据的存储方式，session数据可以保存在数据库、本地缓存中等等。

2.2.1 数据库

按如下方式配置 SESSION_ENGINE，Session 就存储在数据库中，这也是 SESSION_ENGINE 的默认值。

SESSION_ENGINE='django.contrib.sessions.backends.db'

如果存储在数据库中，还需要在项 INSTALLED_APPS 中安装 Session 应用，因为会用到 Session 应用里面的模型类。

2.2.2 本地缓存

存储在本机内存中，如果丢失则不能找回，比数据库的方式读写更快。

SESSION_ENGINE='django.contrib.sessions.backends.cache'

2.2.3 混合存储

优先从本机内存中存取，如果没有则从数据库中存取。

SESSION_ENGINE='django.contrib.sessions.backends.cached_db'

2.3 配置cache为Redis

在redis中保存session，需要引入第三方扩展，我们可以使用django-redis来解决。

网络连接

1） 安装扩展

pip install django-redis

2）配置

在settings.py文件中做如下设置

CACHES = {
    'default': {
        'BACKEND': 'django_redis.cache.RedisCache',
        'LOCATION': 'redis://127.0.0.1:6379/1',
        'OPTIONS': {
            'CLIENT_CLASS': 'django_redis.client.DefaultClient',
        }
    }
}
SESSION_ENGINE = 'django.contrib.sessions.backends.cache'
SESSION_CACHE_ALIAS = 'default'

2.4 Session操作

通过 HttpRequest 对象的 session 属性进行会话的读写操作。

1） 以键值对的格式写session。

request.session['键']=值

2）根据键读取值。

request.session.get('键',默认值)

3）清除所有session，在存储中删除值部分。

request.session.clear()

4）清除session数据，在存储中删除session的整条数据。

request.session.flush()

5）删除session中的指定键及值，在存储中只删除某个键及对应的值。

del request.session['键']

6）设置session的有效期

request.session.set_expiry(value)
  • 如果value是一个整数，session将在value秒没有活动后过期。
  • 如果value为0，那么用户session的Cookie将在用户的浏览器关闭时过期。
  • 如果value为None，那么session有效期将采用系统默认值， 默认为两周，可以通过在settings.py中设置SESSION_COOKIE_AGE来设置全局默认值。
