6.5.查询集QuerySet

查询集QuerySet

学习目标

  • 理解 QuerySet 的概念和2大特性
  • 理解 QuerySet 和 objects 对象的关系
  • 熟练使用 QuerySet 索引和切片
  • 熟练使用 Paginator 进行数据的分页获取

一、概念

QuerySet 查询集，也称查询结果集，表示从数据库中获取的对象集合，QuerySet 不是简单的列表。

  • QuerySet 类型对象可以调用 all、filter、exclude、order_by 这些方法
  • all、filter、exclude、order_by 这些方法的返回值是 QuerySet 类型对象
    • all()：返回所有数据
    • filter()：返回满足条件的数据
    • exclude()：返回满足条件之外的数据
    • order_by()：对结果进行排序

QuerySet 可以通过链式调用对多个查询条件进行组合

>>> books = BookInfo.objects.filter(readcount__gt=30).order_by('pub_date')
>>> books
<QuerySet [<BookInfo: 天龙八部>, <BookInfo: 雪山飞狐>]>
  • exists()：判断查询集中是否有数据，如果有则返回True，没有则返回False。

二、两大特性

1）惰性执行

创建查询集不会访问数据库，直到调用数据时，才会访问数据库，调用数据的情况包括迭代、序列化、与if合用

例如，当执行如下语句时，并未进行数据库查询，只是创建了一个查询集 books

books = BookInfo.objects.all()

继续执行遍历迭代操作后，才真正的进行了数据库的查询

for book in books:
    print(book.name)

2）缓存

使用同一个查询集对象，第一次使用时会发生数据库的查询，Django 会把数据库查询结果缓存下来，再次使用这个查询集时会直接使用缓存的数据，减少了数据库的查询次数。

情况一：如下是两个查询集，无法重用缓存，每次查询都会与数据库进行一次交互，增加了数据库的负载。

from book.models import BookInfo
[book.id for book in BookInfo.objects.all()]
[book.id for book in BookInfo.objects.all()]

情况二：经过存储后，可以重用查询集，第二次使用缓存中的数据。

books=BookInfo.objects.all()
[book.id for book in books]
[book.id for book in books]

三、QuerySet和objects的关系

  • objects 是 Manager 类型对象

  • Manager 继承自 BaseManagerFromQuerySet 类

  • BaseManagerFromQuerySet 类是 django.db.models.manager.BaseManager 和 django.db.models.query.QuerySet 共同构建的一个类
    • BaseManagerFromQuerySet 是 BaseManager 的子类
    • BaseManagerFromQuerySet 拷贝了 QuerySet 类的所有公共方法

四、限制查询集

可以对查询集进行取下标或切片操作，等同于 sql 中的 limit 和 offset 子句。

注意：不支持负数索引。

对查询集进行切片后返回一个新的查询集，不会立即执行查询。

如果获取一个对象，直接使用[0]，等同于[0:1].get()，但是如果没有数据，[0]引发IndexError异常，[0:1].get()如果没有数据引发DoesNotExist异常。

示例：获取第1、2项，运行查看。

>>> books = BookInfo.objects.all()[0:2]
>>> books
<QuerySet [<BookInfo: 射雕英雄传>, <BookInfo: 天龙八部>]>
