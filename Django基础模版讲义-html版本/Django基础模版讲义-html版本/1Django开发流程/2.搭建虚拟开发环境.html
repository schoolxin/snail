2.搭建虚拟开发环境

搭建虚拟开发环境

学习目标

  • 安装虚拟环境管理工具
  • 创建、切换、退出、删除虚拟环境

一、虚拟环境简介

1.1 什么是虚拟环境

虚拟环境，类似于又安装了一个的 python 解释器，新安装的 python 解释器和原来的 python 解释器同时存在，各有自己的 python 包文件夹 ，也就是python解释器安装目录中的 lib/site-packages/，这样两个 python 解释器安装 python 包时相互独立、互不干扰，独立管理和使用 python 包。但虚拟环境不是真正的安装了一个 python 解释器，而是通过工具模拟出来这种效果，这种工具叫做虚拟环境管理工具。

虚拟环境管理工具，一种对虚拟环境进行创建、切换、退出、删除等管理操作的工具。

1.2 为什么需要虚拟环境

问题：当我们在同一个电脑上，同时开发多个项目的时候，如果用到了同一个解释器下面、同一个包的不同版本，就会发生冲突，因为在同一个解释器里面，同一包只能存在一个版本，当我们安装一个包的另外一个版本的时候，pip 会先卸载已经安装过的版本，再安装新的版本。

解决方案：虚拟环境

虚拟环境可以根据已经安装的 Python 解释器，搭建新的、独立的 python 运行环境, 相当于把 python 解释器复制了一份放在另外一个目录下面（Linux下只创建一个符号链接），不同解释器下面安装的包互不干扰，可以使得每个项目的使用一个自己的、独立的解释器环境，与其它项目互不影响。

二、安装虚拟环境管理工具

在 终端或者命令行中使用下面的命令安装虚拟环境管理工具

# mac 或者 linux
sudo pip install virtualenv
sudo pip install virtualenvwrapper

# windows
pip install virtualenv
pip install virtualenvwrapper-win

linux 下安装完虚拟环境后，如果提示找不到mkvirtualenv命令，须配置环境变量：

# 1、创建虚拟环境工作文件夹
mkdir $HOME/.virtualenvs

# 2、修改~/.bashrc，设置WORKON_HOME环境变量和开机启动脚本
export WORKON_HOME=$HOME/.virtualenvs
source /usr/local/bin/virtualenvwrapper.sh

#3、运行
source ~/.bashrc

三、使用虚拟环境

3.1 创建虚拟环境

使用默认 Python 解释器创建虚拟环境

mkvirtualenv 虚拟环境名称
例 ：
mkvirtualenv py_django

指定 Python 解释器创建虚拟环境

mkvirtualenv -p python3 虚拟环境名称
例 ：
mkvirtualenv -p python3 py3_django

[!TIP|style:flat]

  1. 创建虚拟环境需要联网
  2. 创建成功后, 会自动工作在这个虚拟环境上
  3. 工作在虚拟环境上, 提示符最前面会出现 “虚拟环境名称”

3.2 查看虚拟环境列表:

workon

3.3 切换到虚拟环境 :

workon 虚拟环境名称
# 使用 py3_django 的虚拟环境
workon py3_django

3.4 退出虚拟环境 :

deactivate

3.5 删除虚拟环境 :

rmvirtualenv 虚拟环境名称

# 例 ：删除虚拟环境py3_django
先退出：deactivate
再删除：rmvirtualenv py3_django

3.5 在虚拟环境中安装工具包

切换到虚拟环境后，使用 pip 命令安装工具包

pip install 包名称

虚拟环境中，python 包安装在下面目录中

{虚拟环境workon目录}/{虚拟环境名字}/lib/{python解释器版本}/site-packages/

比如

~/.virtualenvs/py3_flask/lib/python3.5/site-packages

例 : 安装django-1.11.11的包

pip install django==1.11.11

查看虚拟环境中安装的包 :

pip list
