Django简介

学习目标

  • 理解 HTTP 服务器的工作流程
  • 了解 Django 的诞生和特点
  • 【重点】理解 MVT 设计模式

一、HTTP服务器工作流程

打开 www.taobao.com，登录之后来到首页

1.1 从 HTTP 角度看

  • HTTP客户端：通过 sokect 和服务器建立 tcp 连接，发送 http 请求

  • HTTP服务器：通过 sokect 和客户端建立 tcp 连接，发送 http 响应

1.2 从HTTP服务器内部看

WSGI服务器：

  • 负责通过 socket 接受 http 请求数据
  • 解析 http 请求数据
  • 按 http 协议格式拼接 http 响应数据

WSGI应用：

  • 产生 http 响应数据

1.3 从WSGI应用内部看

Web 框架：

  • URL 路由：根据 URL 路径调用对应的 web 应用处理 http 请求
  • 中间件数据处理：在 web 应用处理 http 请求数据前，修改处理 http 请求数据，比如校验 cookie 数据；在 web 应用产生 http 响应数据后，返回给 WSGI 服务器之前，修改处理 http 响应数据，比如在 cookie 中添加请求头。

Web 应用：

  • 根据 http 请求数据，处理相应的逻辑，产生 http 响应数据，比如登录这个功能，用户应用会获取 http 请求里面的用户名和密码，然后去数据库里面查询用户名是否存在，密码是否一致，如果用户名存在，并且密码一致，就返回用户登录成功的页面数据，否则返回用户登录失败的页面数据。

1.4 WSGI协议

WSGI 协议规定了Python Web 服务器和 Web应用之间通过什么方式相互调用。

WSGI 协议规定，WSGI 应用必须有一个可以产生 HTML 页面数据的函数，WSGI 服务调用该函数获取响应数据。

def application(environ, start_response):
    start_response('200 OK', [('Content-Type', 'text/html')])
    return [b'<h1>Hello, web!</h1>']
  1. 函数有两个参数，environ 里面保存的是 http 请求的相关信息，start_response 是一个回调函数，用来设置 http 响应的响应头
  2. 函数的返回值是 http 的响应体

1.5 Web程序框架

不用重复编写不同 Web 应用中相同的这一部分代码，只需关心 Web 应用核心的业务逻辑实现。

Python 下有许多款不同的 Web 框架，比如 Django、Flask、Tornado、Bottle、Web.py 等等。

二、Django框架简介

2.1 Django的诞生

Django，发音为[`dʒæŋɡəʊ]，是用 python 语言写的开源web开发框架，并遵循 MVC 设计。在内容管理系统、社交网络、科学计算平台等等，许多成功的网站和APP都是基于 Django 开发的，比如 领英、Instagram、美国国家地理、open stack 等等。Django 最早是劳伦斯出版集团为了开发以新闻内容为主的网站，而开发出来了这个框架，于2005年7月在BSD许可证下发布。这个名称来源于比利时的爵士音乐家 DjangoReinhardt，他是一个吉普赛人，主要以演奏吉它为主，还演奏过小提琴等。由于 Django 在近年来的迅速发展，应用越来越广泛，被著名IT开发杂志 SDTimes 评选为 2013SDTimes100，位列"API、库和框架"分类第6位，被认为是该领域的佼佼者。

Django 的主要宣传点是简便、快速的开发数据库驱动的网站。它强调代码复用，许多功能以组件的形式已经存在，比如 用户身份验证、内容管理等等，组件可以很方便的以"插件"形式服务于整个框架，Django 有许多功能强大的第三方插件，你也可以很方便的开发出自己的工具包。这使得Django 具有很强的可扩展性。它还强调快速开发和 DRY(DoNotRepeatYourself)原则。

2.2 Django框架特点

2.2.1 重量级框架

Django原生提供了众多的功能组件，让开发更简便快速。

  • 提供项目管理的自动化脚本工具

三、MVT模式

3.1 MVC模式

有一种软件架构设计模式叫 MVC，其核心思想是分工、解耦，让不同的代码块之间降低耦合，增强代码的可扩展性和可移植性，实现向后兼容。

MVC 的全拼为 Model-View-Controller，最早由 TrygveReenskaug 在1978年提出，是施乐帕罗奥多研究中心(Xerox PARC)在20世纪80年代为程序语言 Smalltalk 发明的一种软件设计模式，是为了将传统的输入（input）、处理（processing）、输出（output）任务运用到图形化用户交互模型中而设计的。随着标准输入输出设备的出现，开发人员只需要将精力集中在业务逻辑的分析与实现上。后来被推荐为 Sun 公司 Java EE 平台的设计模式，并且受到越来越多的开发者的欢迎，并用在其他语言和平台上面。现在虽然不再使用原来的分工方式，但是这种分工的思想被沿用下来，广泛应用于软件工程中，是一种典型并且应用广泛的软件架构模式。后来，MVC 的思想被应用在了 Ｗeb 开发方面，被称为 Ｗeb MVC 框架。

比如用户登录，浏览器把用户名和密码提交给了服务器，Controller 从请求对象中获取 用户名和密码，然后Controller 通过 Model 查询用户名在数据库表中是否存在，如果存在，再查询改用户的密码是否和数据库中保存的密码一致，如果相同，那么 Controller 通过 View 构建登录成功的 HTML 页面，否则，构建登录失败的 HTML 页面，最后 Controller 把构建好的 HTML 页面返回给浏览器。

  • M全拼为Model，主要封装对数据库层的访问，对数据库中的数据进行增、删、改、查操作。
  • V全拼为View，用于封装生成图形界面，生成页面展示的html内容。
  • C全拼为Controller，用于接收请求，处理业务逻辑，与Model和View交互，返回结果。

3.2 Django的MVT模式

  • M全拼为Model，与MVC中的M功能相同，负责和数据库交互，进行数据处理。
  • V全拼为View，与MVC中的C功能相同，接收请求，进行业务处理，返回应答。
  • T全拼为Template，与MVC中的V功能相同，负责封装构造要返回的html。
